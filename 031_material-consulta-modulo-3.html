Cómo usar este material

Las fuentes de consulta de base para el desarrollo de la actividad corresponden a las lecturas sugeridas dispuestas en el aula virtual o cualquier otra fuente que se considere pertinente consultar. A continuación se recopilan las fuentes principales del Módulo 3, organizadas por tipo: bibliografía, documentación oficial y técnica, recursos audiovisuales e interactivos, marco normativo-ético y conjuntos de datos de práctica. El módulo recorre tres familias de implementación, identificadas con una etiqueta de color en cada referencia.

Frameworks — TensorFlow / PyTorch / Keras 3 HPC / GPU — CUDA, cuDNN y cómputo acelerado Pipelines / Despliegue — tf.data, ONNX, exportación

Bibliografía principal

Familia Referencia Aporte al módulo Enlace
Frameworks Goodfellow, I., Bengio, Y., & Courville, A. (2016). Deep Learning. MIT Press. Texto canónico del curso. Fundamenta el porqué matemático que subyace a cada framework: grafos computacionales, diferenciación automática (autograd) y optimización por gradiente que los motores de TensorFlow y PyTorch implementan. deeplearningbook.org
Frameworks Géron, A. (2022). Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow (3.ª ed.). O'Reilly Media. Referencia operativa central del módulo. Cubre la API de alto nivel de Keras, el pipeline tf.data y el guardado/despliegue de modelos en el ecosistema TensorFlow 2.x. Recurso disponible en la biblioteca USTA
Frameworks Stevens, E., Antiga, L., & Viehmann, T. (2020). Deep Learning with PyTorch. Manning Publications. Guía de referencia para PyTorch: tensores, grafos computacionales dinámicos, autograd, módulos (nn.Module) y el pipeline explícito de datos con DataLoader. Recurso disponible en la biblioteca USTA
Pipelines Chollet, F. (2021). Deep Learning with Python (2.ª ed.). Manning Publications. Escrito por el creador de Keras. Aporta la intuición práctica del flujo de trabajo completo —de la construcción del modelo a su entrenamiento, evaluación y despliegue— que sustenta el Proyecto Centinela · Fase 3. Recurso disponible en la biblioteca USTA

Documentación oficial y técnica

En la fase de implementación, la fuente primaria deja de ser el paper seminal y pasa a ser la documentación oficial de cada herramienta. Estas referencias son la verdad operativa del módulo: APIs, formatos de exportación y guías de rendimiento que cambian con cada versión.

FrameworksTensorFlow, PyTorch y Keras 3

Referencia Aporte al módulo Enlace
Keras Team. (2023). Introducing Keras 3.0. Keras. Keras 3 como API multi-backend: el mismo código corre sobre TensorFlow, JAX o PyTorch. Documenta "el puente entre mundos" que permite escribir un modelo agnóstico al motor de cómputo, eje del capítulo de frameworks. keras.io/keras_3
PyTorch Foundation. (2025). torch.export documentation. PyTorch. Documenta torch.export, el mecanismo moderno para capturar un grafo de PyTorch y exportarlo a un formato portable. Pieza clave para llevar un modelo entrenado del cuaderno a producción. docs.pytorch.org/.../export

HPC/GPUCómputo de alto rendimiento: CUDA y cuDNN

Referencia Aporte al módulo Enlace
NVIDIA Developer. (2025). CUDA Deep Neural Network (cuDNN). NVIDIA. cuDNN: la biblioteca de primitivas optimizadas (convolución, pooling, normalización) sobre la que TensorFlow y PyTorch delegan el cálculo en GPU. Explica la base de la aceleración por hardware del módulo. developer.nvidia.com/cudnn
NVIDIA Developer. (2025). CUDA Toolkit Documentation. NVIDIA. CUDA: la plataforma de cómputo paralelo de propósito general sobre GPU. Sustenta el concepto de "la pila de software" y por qué la VRAM es el recurso más escaso al entrenar redes profundas. developer.nvidia.com/cuda-toolkit

PipelinesPipelines de datos, aumento y despliegue

Referencia Aporte al módulo Enlace
TensorFlow Authors. (2023). Better performance with the tf.data API. TensorFlow. Guía oficial del pipeline declarativo tf.data: prefetch, cache y paralelización para que la GPU nunca espere por los datos. Núcleo del capítulo de pipelines de entrada. tensorflow.org/guide/data_performance
Buslaev, A., Iglovikov, V., & Parinov, A. (2020). Albumentations: Fast and flexible image augmentations. Information, 11(2), 125. (AlbumentationsX, v2025). Biblioteca de aumento de datos de alto rendimiento. AlbumentationsX es el sucesor mantenido; se integra al pipeline para generar variabilidad realista sin salir del entorno de cómputo. albumentations.ai
ONNX Community. (2024). ONNX: Open Neural Network Exchange. The Linux Foundation. ONNX, "el pasaporte universal de los modelos": un formato abierto que permite exportar un modelo de un framework e importarlo en otro o en un motor de inferencia. Cierre del capítulo de despliegue. onnx.ai

La documentación versiona. A diferencia de los papers, estas fuentes cambian con cada versión. Verifica siempre que la página de documentación corresponda a la versión instalada en tu entorno (p. ej. la API de torch.export o de Keras 3 puede diferir entre minor releases).

Recursos audiovisuales e interactivos

Herramientas para "tocar" la implementación sin fricción: simuladores en el navegador, documentación práctica del entorno de cómputo y explicaciones visuales en español.

Familia Recurso Descripción y vínculo pedagógico Enlace
Frameworks TensorFlow Playground (Smilkov & Carter, Google) Simulador web para experimentar con redes en tiempo real sin instalar nada: capas, tasa de aprendizaje, funciones de activación y regularización. Primer contacto visual con los hiperparámetros que luego se configuran en código. playground.tensorflow.org
HPC/GPU Google Colab — Preguntas Frecuentes (FAQ) (Google Research) Documentación oficial del entorno de cómputo gratuito con GPU: límites de uso, tipos de aceleradores, sesiones efímeras y buenas prácticas. Lectura imprescindible para entender la metodología "Ingeniería de Trinchera". research.google.com/.../faq
Frameworks Dot CSV (Carlos Santana, YouTube · español) Canal de divulgación en español que explica con claridad visual frameworks, GPU y el flujo de implementación de modelos de Deep Learning. Apoyo en lengua materna para reforzar los conceptos del módulo. youtube.com/c/DotCSV
Pipelines CNN Explainer (Polo Club of Data Science, Georgia Tech, 2020) Visualización interactiva que permite "abrir" una CNN capa por capa (convolución, ReLU, pooling). Útil para razonar qué operaciones aceleran cuDNN y la GPU al ejecutar la inferencia que se despliega. poloclub.github.io/cnn-explainer
Pipelines Transformer Explainer (Polo Club of Data Science, Georgia Tech, 2024) Ejecuta un modelo tipo GPT-2 directamente en el navegador, visualizando la atención multi-cabeza en tiempo real. Ejemplo de cómo un modelo entrenado se sirve y se ejecuta de cara al usuario final. poloclub.github.io/transformer-explainer

Estrategia de cómputo (Colab + Kaggle). Combina los dos entornos gratuitos para maximizar GPU: Google Colab para iteración rápida y prototipado, y Kaggle Notebooks con su cuota predecible de ~30 h de GPU + ~30 h de TPU por semana (≈60 h) para entrenamientos largos. Así, si un entorno se queda sin sesión, el otro mantiene el proyecto en marcha sin pagar nada: pura "Ingeniería de Trinchera".

Implicación ética del módulo. La fase de implementación y despliegue es donde un modelo deja de ser un experimento y empieza a afectar a personas reales. La trazabilidad (qué versión del modelo, con qué datos, sobre qué hardware) y la reproducibilidad dejan de ser un lujo de ingeniería y pasan a ser un requisito ético. La auditoría del Proyecto Centinela · Fase 3 exige documentar el entorno de cómputo, las versiones de los frameworks y el formato de exportación, en línea con el Marco Ético para la IA en Colombia.

Datasets y herramientas de práctica

Política de datos del proyecto. Kaggle Notebooks está permitido como entorno de cómputo, pero los datasets y competencias de Kaggle NO se permiten como fuente de datos de la actividad: son retos ya resueltos y con datos demasiado limpios. El proyecto exige fuentes reales y públicas (se admiten fuentes internacionales).

RoCoLe — Parraga-Alava, J. et al. (2019). RoCoLe: A robusta coffee leaf images dataset (v2). Mendeley Data. Dataset del escenario de referencia del Proyecto Centinela. En la Fase 3 las imágenes de hojas de café Robusta (sanas / con roya) alimentan un pipeline reproducible con aumento de datos (AlbumentationsX) y un modelo exportable a producción. data.mendeley.com
FashionMNIST — Xiao, H., Rasul, K., & Vollgraf, R. (2017). Zalando Research. Dataset de referencia ligero (70.000 imágenes de prendas en 28×28). Ideal como banco de pruebas para validar el pipeline completo (tf.data / DataLoader, entrenamiento en GPU, guardado y exportación a ONNX) antes de escalar al dato real del proyecto. github.com/zalandoresearch
Google Colab + Kaggle Notebooks Entornos de cómputo gratuitos con GPU. Toda la implementación práctica del módulo se ejecuta en estos entornos según la metodología "Ingeniería de Trinchera". Kaggle Notebooks está permitido como entorno de cómputo, pero sus datasets y competencias no se admiten como fuente de datos de la actividad (véase la alerta superior). colab.research.google.com
kaggle.com/code
ONNX Runtime + AlbumentationsX ONNX Runtime: motor de inferencia multiplataforma para ejecutar el modelo exportado a ONNX con baja latencia. AlbumentationsX: biblioteca de aumento de imágenes de alto rendimiento para el pipeline de entrenamiento. Herramientas operativas del cierre de despliegue del módulo. onnxruntime.ai
albumentations.ai

Atención a la licencia de AlbumentationsX. El sucesor mantenido de Albumentations, AlbumentationsX, se distribuye bajo licencia AGPL-3.0 (con opción de licencia comercial). La AGPL es "copyleft fuerte" y puede imponer obligaciones de apertura del código si se usa en un servicio en producción. Para el trabajo académico del curso no hay problema, pero documéntalo en la bitácora y, si despliegas un servicio real, revisa la compatibilidad de licencias (la versión clásica de Albumentations conserva su licencia MIT, más permisiva).
